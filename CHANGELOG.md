## 版本定位

`v1.3.1` 是兼容性、安全性与稳定性修复版本，不新增必须迁移的公开配置项。现有私聊、群聊、会话覆写和群聊批次配置可以继续使用。最低 AstrBot 版本仍为 `4.8.0`。

更完整的 Fork 差异、维护边界和升级说明请参阅 [Fork 维护与版本说明](docs/fork-maintenance.md)。

## 核心兼容性修复

### 修复 OutputPro 等输出增强插件丢失第一段

- 修复主动消息使用的合成事件只有“发送记录”、没有真正发送能力的问题。
- 装饰插件在 `on_decorating_result` 中调用 `event.send()` 提前发送分段时，现在会通过目标平台真实投递，不再出现“日志中有第一段，聊天窗口没有第一段”的情况。
- 主动消息合成事件标记为 `ResultContentType.LLM_RESULT`，并设置 `action_type=proactive`，使“仅处理 LLM 回复”的输出增强步骤能够正确识别主动消息。
- 保持 AstrBot `>=4.8.0` 兼容，不依赖新版才存在的消息链派生接口。

### 恢复完整的装饰发送语义

- 装饰器提前发送的分段和最后剩余分段均按原顺序真实发送，装饰钩子每轮只执行一次，不会递归触发自身。
- 正确处理 `stop_event()`、`continue_event()`、`clear_result()`、空消息链和仅包含空白 `Plain` 的拦截结果。
- 装饰器已经发送部分内容后再抛出普通异常时，不再把完整原文作为兜底重复发送。
- 保留消息链的 T2I、Markdown 和消息类型元数据；装饰器登记的临时文件会在全部提前发送与末段发送完成后再清理。
- 不额外触发 `OnAfterMessageSentEvent`，避免主动插件自身缓存和群聊沉默计时重复执行。

### 使用真实发送结果驱动后续逻辑

- 新增结构化发送结果，分别记录尝试、成功、失败、停止和拦截状态。
- 只要至少一个物理分段真实送达，本轮即视为已送达；部分失败只记录告警，不自动重发整句，避免用户收到重复内容。
- 全部分段失败时才进入原有失败调度；纯拦截不会写缓存、对话历史、未回复计数或群聊沉默计时。
- 平台明确返回 `False` 时视为失败；兼容成功时返回 `None` 的旧平台接口。
- 优先按平台实例 ID 精确匹配；同名或重复实例无法唯一确定时拒绝发送，避免消息发到错误机器人。
- 每个成功物理分段只补写一条平台流水；运行时缓存和 LLM 历史仍只保存一次完整逻辑回复。

## 配置与 AstrBot 兼容修复

- 新增统一配置处理器，对群聊批次、会话列表、调度、自动触发、上下文、运行时缓存、TTS 和分段回复配置进行集中校验。
- Web 管理端和会话覆写接口采用严格模式：非法根类型、子字段类型、数值范围、枚举、正则或间隔关系会返回 HTTP 400，并保持原配置不变。
- 启动和运行时采用宽容模式：旧配置中的 `null`、字符串批次、坏数字或坏嵌套对象会被跳过或回退为安全默认值，不再导致插件初始化、状态页或发送流程崩溃。
- 群聊批次缺省字段继续继承全局群聊配置，不会被处理器默认值意外覆盖；首个匹配批次仍具有优先级。
- 静态 Schema 将 `group_batches` 保持为 AstrBot 4.8 能识别的普通 `list`；新版 AstrBot 在内存中按能力升级为对象模板列表。
- 配置路由补齐 `PrivateMessage` 和 `GuildMessage`，群聊判断统一按消息类型精确识别，不再被平台名称中的 `group` / `guild` 字样误导。

## 历史、缓存与调度修复

- 修复主动直写缓存与 AstrBot 发送后事件对同一条 Bot 消息重复记录的问题，同时保留同一来源、不同消息 ID 的真实重复消息。
- 修复主动消息历史幂等判断顺序错误导致相同 Prompt / 回复对重复写入的问题。
- 修复对话末尾已有孤立 assistant 消息时，主动 Prompt 与回复无法正确组成一轮历史的问题。
- 保持原始 UMO、规范化 UMO 和当前 conversation 映射一致，避免平台别名变化造成上下文断裂。
- 群聊沉默计时只在确有消息送达时重置；插件关闭期间禁止新任务、计时器和发送继续产生。

## Web 管理端安全修复

- 修复设置密码后仍可使用固定 `no-auth` 令牌绕过 HTTP 与 WebSocket 鉴权的问题。
- 非对象或非字符串密码配置现在安全锁闭，不会因配置损坏退化为无密码访问。
- 密码新增、修改、清空或从其他入口热更新时，立即废除旧 Token 并关闭旧 WebSocket。
- WebSocket Token 不再通过 URL query 参数传输，改为连接建立后的首帧认证。
- WebSocket 在处理入站消息和发送广播前都会重新同步鉴权状态；HTTP 401 或 WebSocket 1008 会让前端清理失效令牌并返回登录流程。
- Web 管理端批次和会话配置更新改为“先完整校验、再一次性替换”，避免错误请求保存一半配置。

## 隐私、安全日志与 Fork 策略

- 本 Fork 已彻底移除匿名遥测、远程错误上报、实例 ID、心跳任务和全局 asyncio 异常处理器；配置、文档和生命周期中的遥测入口同步删除。
- 新增统一安全异常日志，只记录固定错误编号、模块语义和经过白名单确认的异常类型。
- 不再记录异常正文、完整 traceback、Prompt、聊天正文、UMO、目标 ID、平台实例 ID、conversation ID、会话备注或批次名称。
- 安全日志防御伪造异常类名、`__module__`、`__bool__`、`__hash__` 和 `__eq__`，避免第三方异常反过来突破日志边界。

## 生命周期与资源清理

- 插件终止时先阻止新任务，再取消群聊与自动触发计时器，随后停止 Web 管理端和通知中心，最后清理通用后台任务与持久化状态。
- 通知轮询任务立即取消时会正确处理 `CancelledError`，清空任务句柄并继续保存通知缓存。
- TTS 临时文件在发送成功、失败和取消路径中都会精确清理；支持普通路径以及大小写不敏感的本机 `file://localhost` URI，不会删除远程 URL 或网络文件 URI。

## 测试与验证

- 当前 AstrBot 插件全量测试：`126 passed`。
- 当前 AstrBot 真实插件管理器加载：`5 passed`。
- AstrBot 4.8 兼容环境：`99 passed, 24 skipped`。
- OutputPro 时间线测试：`29 passed`。
- 真实 OutputPro 分段链确认第一段、后续分段和末段按顺序送达，装饰钩子只执行一次。
- Ruff 静态检查、Ruff 格式、Python 编译、Schema JSON、前端 JavaScript 语法、`git diff --check` 和 LF 行尾检查全部通过。

# 2026/03/30 v1.2.0

经历了大半个月的高强度开发，新版本的主动消息插件终于和大家见面了！
首先非常感谢大家对本插件的喜爱，插件的 Star 数一个月就翻了将近一倍，让我非常的惊喜，也敦促着我去进一步完善这个插件。
那么现在，你可以通过插件原生的 WebUI 对主动消息插件进行高效的配置与日常使用，新版本的个性化配置请通过插件自带的 Web 端进行配置。
在正式使用新版本插件前，请仔细阅读以下说明：

> [!IMPORTANT]
>
> 由于新版本的重构需要，更新后您需要以 `UMO` 的格式，重新配置需要接收主动消息的会话列表。
>
> 完整的 UMO 可通过 `/sid` 指令快捷获取。格式: `{platform_name}:{message_type}:{session_id}`
>
> `{platform_name}` 就是你在 WebUI 中的 `机器人` 一栏中自定义的机器人名称。`{message_type}` 可选参数为 `GroupMessage` 和 `FriendMessage`
>
> 例如：如果你之前在**私聊**列表中填写的是 `123456789`，自定义的机器人名称 (平台名称) 为 `default`，请改为 `default:FriendMessage:123456789`。
> 如果你之前在**群聊**列表中填写的是 `987654321`，自定义的机器人名称 (平台名称) 为 `default`，请改为 `default:GroupMessage:987654321`。
>
> 使用 QQ 官机时，注意不要像个人号那样填写 QQ 号，应填写 UID，可使用指令 `/sid` 获取，格式类似 `4C011A2B3D4C5E6F9F8E7D6C5B4A3210`。
>
> 使用个人微信需要升级到最新的手机微信版本：`iOS >= 8.0.70`，`Android >= 8.0.69`，并确保微信中包含 `ClawBot` 插件
>
> 如果更新到新版本发现自己的某些配置项或自定义提示词被重置，可以参考插件数据文件夹下的配置快照与提示词备份来辅助恢复。
>
> 如遇到无法正常使用的问题，欢迎通过 Issue 等方式向开发者反馈。
